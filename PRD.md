# 波动贡献分析工具 — 需求文档

## 一、项目概述

一个网页工具，用于对结构化数据进行**波动贡献分析（Volatility Contribution / Structural Decomposition Analysis）**。用户上传表格文件，选择分析维度、时间字段和指标字段，系统自动计算各维度的波动贡献值及贡献度，并以表格形式展示结果。

## 二、核心功能

### 1. 文件上传
- 支持 CSV、Excel（.xlsx/.xls）格式
- 首行自动识别为表头
- 上传后预览数据前 10 行

### 2. 字段选择
- **分析维度**：选择一列或多列作为分组维度（如产品类型、地区、渠道等）
- **时间字段**：选择一列作为时间标识（列内需有"基期"和"本期"两个不同值，用户指定哪个是基期、哪个是本期）
- **指标字段**：选择一列数值列作为分析指标（如单价、利润率、转化率等）
- **分母字段**：选择一列数值列作为分母（如销量、流量、总用户数等），用于计算各维度占比

### 3. 分析结果表

每一行代表一个维度取值组合，包含以下列：

| 列名 | 说明 | 计算方式 |
|------|------|----------|
| 维度 | 分组维度的取值 | 原样展示 |
| 本期分母占比 | 当前周期该维度占总分母的比例 | 本期维度分母 / 本期总分母 |
| 基期分母占比 | 基期该维度占总分母的比例 | 基期维度分母 / 基期总分母 |
| delta分母占比 | 分母占比的变化 | 本期分母占比 − 基期分母占比 |
| 本期指标值 | 该维度在当前周期的指标值 | 本期指标值（加权平均或直接取值） |
| 基期指标值 | 该维度在基期的指标值 | 基期指标值 |
| delta指标值 | 指标值的变化 | 本期指标值 − 基期指标值 |
| 组内贡献值 | 指标自身变化带来的贡献 | 基期分母占比 × delta指标值 |
| 组间贡献值 | 结构变化带来的贡献 | delta分母占比 × 基期指标值 |
| 交叉贡献值 | 指标与结构交互变化的贡献 | delta分母占比 × delta指标值 |
| 波动贡献值 | 该维度对总波动的净贡献 | 组内 + 组间 + 交叉 |
| 贡献度 | 该维度波动贡献占总波动的百分比 | 波动贡献值 / Σ\|波动贡献值\| × 100% |

### 4. 汇总行
- 表格底部显示各数值列的合计行
- 验证：Σ波动贡献值 = 总指标变动（本期总指标 − 基期总指标）

## 三、算法说明

### 核心公式

设有 n 个维度分组，指标加权平均值：X = Σ(w_i × x_i)

其中 w_i 为第 i 组的分母占比，x_i 为第 i 组的指标值。

**总变动**：ΔX = X₁ − X₀ = Σ(w_i₁ × x_i₁) − Σ(w_i₀ × x_i₀)

**三维分解**（对每个维度 i）：

| 贡献类型 | 公式 | 含义 |
|----------|------|------|
| 组内贡献 | w_i₀ × (x_i₁ − x_i₀) | 指标本身变化，结构不变 |
| 组间贡献 | (w_i₁ − w_i₀) × x_i₀ | 结构变化，指标不变 |
| 交叉贡献 | (w_i₁ − w_i₀) × (x_i₁ − x_i₀) | 指标和结构同时变化的交叉效应 |

**验证公式**：Σ(组内 + 组间 + 交叉) = X₁ − X₀ = ΔX

## 四、交互流程

```
[上传文件] → [预览数据] → [选择维度列] → [选择时间列 + 标记基期/本期]
    → [选择指标列] → [选择分母列] → [点击分析] → [展示结果表]
```

## 五、非功能需求
- 纯前端运行，数据不上传服务器
- 支持至少 10 万行数据处理
- 结果表支持按列排序
- 结果表支持导出为 Excel/CSV
- 界面简洁，响应式布局，适配桌面和移动端

## 六、技术方案建议
- 前端框架：React + TypeScript + Vite
- 表格解析：SheetJS (xlsx) 用于 Excel，PapaParse 用于 CSV
- 表格展示：简单的 HTML table 或轻量表格组件
- 部署：纯静态网站，可部署到 Netlify / GitHub Pages
